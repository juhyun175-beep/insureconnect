### 한 줄 진단

**성장은 이미 터졌다. 문제는 수익화 인프라가 전부 "설치만 되고 전원이 꺼져 있다"는 것.**

- 30일에 회원 78%가 새로 들어왔다 → 유입은 검증 끝. 오픈채팅+네이버 조합이 작동한다.
- 745,000원의 주문이 잡혔는데 전부 입금대기 = 상품성은 검증됐고 **결제 단계에서 새고 있다.** (무통장 수기 입금의 한계)
- 네이버 유입 상위 랜딩 1~12위가 전부 `/company/{보험사}` 전산 페이지 → **설계사가 매일 검색하는 실무 키워드를 이미 점령했다.** 이 트래픽에 수익 지점이 안 붙어 있다.
- 토스페이먼츠(`functions/api/payments/*`), 주간 다이제스트(`cron/weekly-digest`), 포인트샵, 쿠팡 제휴 — 코드는 다 있는데 꺼져 있거나 비어 있다.

### 전략 골격

```
Phase 1 (7~9월)   있는 걸 켠다        → 첫 확정매출, 월 100만원
Phase 2 (10~12월) 성장과 매출 동시    → MAU 800, 월 300~500만원
Phase 3 (2027~)   구독·SaaS 전환      → 월 1,000만원+, 플랫폼 사업화
```

## Phase 1 — 단기 매출 (7~9월): "있는 걸 켠다"

목표: **확정매출 0원 → 월 100만원.** 신규 개발 최소, 활성화 위주.

### 1-1. 입금대기 745,000원부터 회수 — 토스페이먼츠 ON 🔥 최우선

- **왜**: 주문 5건이 전부 `pending_payment`. 상품(모임 광고)은 팔리는데 무통장 수기 입금에서 이탈. 결제 마찰 = 지금 가장 싼 매출 레버.
- **뭘**: 광고 주문 생성 직후 토스 결제창 연결(카드/간편결제). 무통장은 백업으로 유지.
- **어떻게**:
  1. `functions/api/payments/checkout.js`·`confirm.js` 이미 존재 → `ad_orders` 생성 플로우(`_lib/orders.js`)에 결제 URL 연결만 추가.
  2. 미결제 주문 자동 리마인더: 24시간·72시간 경과 시 카톡 메모(`sendMemoToMember` 재사용) "입금 확인이 안 됐습니다" 발송.
  3. 기존 대기 5건은 수동으로 개별 카톡 → 즉시 회수 시도.
- **측정**: 주문→결제완료 전환율 (목표 70%+). `ad_orders.status` 파이프라인을 admin 북극성 패널에 추가.

### 1-2. 모임(meetup) 광고 = 검증된 첫 상품 → 정식 상품화

- **왜**: 유일하게 실제 주문이 발생한 상품. 수요 검증 끝.
- **뭘**: 가격표 공개 페이지 + 상품 등급제. 예: 일반 등록(무료) / 홈 노출 7일(10만) / 홈+카톡 다이제스트 포함(20만) / 긴급·상단고정(30만).
- **어떻게**: `ad_orders`가 이미 ad_type·쿠폰·가격 구조를 지원 → 등급별 가격 상수 + 홈 노출 슬롯 렌더만 추가. 기존 할인권(`coupon_templates`) 시스템으로 첫 구매 할인 프로모션.
- **측정**: 월 광고 주문 건수·객단가. 목표 월 4건 × 15만원 = 60만원.

### 1-3. 카톡 254명 = 방송 채널 → 주간 다이제스트 ON + 광고 슬롯

- **왜**: 옵트인 65%는 업계 최상급 자산. 인프라 완성돼 있는데 발송 게이트가 꺼져 있음(`DIGEST_SEND_ENABLED` OFF). 재방문·광고 지면을 동시에 만드는 한 수.
- **뭘**: 주 1회 다이제스트 실발송 시작. 다이제스트 하단에 "이번 주 파트너" 광고 1슬롯 판매(손해사정사·세무사·GA·교육업체 대상).
- **어떻게**:
  1. `GROWTH_NEXT.md` 9번 절차 그대로: `CRON_SECRET` 등록 → 외부 cron(cron-job.org 무료) 주 1회 POST → DRY 미리보기 검증 → `DIGEST_SEND_ENABLED=1`.
  2. 광고 슬롯은 다이제스트 본문 끝에 텍스트+링크 1줄 — 개발 거의 0, `ad_orders`에 ad_type='digest' 추가.
- **측정**: 발송→클릭률, 발송일 재방문 스파이크. 슬롯 판가 회당 5만 × 월 4회 = 20만원.

### 1-4. 네이버 유입 1등 지면(전산 페이지)에 수익 지점 부착

- **왜**: 네이버 유입 2,003건이 전부 `/company/*`로 꽂히는데 이 페이지에 매출 연결 고리가 약함. `GROWTH.md` 5축 중 '수익' 축이 정확히 비어 있는 곳.
- **뭘**: 전산 페이지 하단·사이드에 ① 렌트카/통신 견적 CTA ② 담보 빠른조회 진입 ③ 회원가입 게이트(청구서류 즐겨찾기 등) 배치. AdSense 자동광고 슬롯 점검.
- **어떻게**: `/company/[slug]` SSR 템플릿에 공용 CTA 블록 1개 삽입 + `trackCardClick` 부착. 견적 모달은 기존 것 재사용.
- **측정**: 전산 페이지→견적 문의 전환. 현재 월 1건 → 목표 월 8건. 렌트카 성사 커미션 건당 10~30만원 가정 시 월 20~60만원.

### 1-5. 채용공고 공급 늘리기 (파는 건 그다음)

- **왜**: 공고 4건뿐 — 지면이 비어 있으면 유료 상단노출(feature)을 팔 수 없다. Google Jobs SEO는 이미 완비(sitemap 165 URL, JobPosting 스키마).
- **뭘**: 3개월간 "공고 등록 무료" 캠페인 → 오픈채팅·다이제스트로 조직장에게 직접 권유 → 공고 20건+ 확보 후 feature 유료화 재개.
- **어떻게**: 개발 0. 운영 액션. 등록 시 `feature_credit` 1회 무료 제공(필드 이미 존재)으로 유료 기능 맛보기.
- **측정**: 공고 수 4→20, 공고 페이지 조회, Google Jobs 노출.

## Phase 2 — 성장+매출 균형 (10~12월)

목표: **MAU 261 → 800 · 월 매출 300~500만원.** 성장 레버와 수익 레버를 짝지어 돌린다.

### 2-1. 구글 SEO 개척 (성장 레버 ①)

- **왜**: 네이버 30% vs 구글 0.8%. 네이버에서 검증된 콘텐츠가 구글에선 존재감 0 = 가장 큰 무료 트래픽 여백. 실손 5세대 전환, 요양실손 등 2026 하반기 제도 변화 = 검색 수요 급증 타이밍.
- **뭘**: SEO 글 120→250편 확장. 우선순위: 수술코드·질병코드·인수사례(경쟁 적고 설계사 검색량 꾸준한 롱테일) + 제도 변화 시사 글.
- **어떻게**: 카테고리 인프라(`ic_seo_posts` 12종)·JSON-LD·sitemap 자동화 완비 → 글 생산만 하면 됨. AI 초안 → 운영자 검수 주 5편 루틴. Search Console 등록·색인 요청 자동화(`scripts/`에 추가). 편당 조회 8회는 내부링크·제목 리라이트로 개선(GROWTH_NEXT 14번 연장).
- **측정**: 구글 유입 55→월 1,000, SEO 글 누적 조회 937→10,000.

### 2-2. 삼따AI를 성장 엔진으로 (성장 레버 ②)

- **왜**: 사례 897건 = 전국 어디에도 없는 데이터. 그런데 30일 사용 78회로 저활용. AI 답변 공유 = 추천 바이럴 루프(v2.18.0)가 이미 구현돼 있는데 굴러가지 않음.
- **뭘**:
  1. 게이트 재설계: 비회원 1회 무료 체험 → 가입 시 3회/일 → **사례 1건 기여 시 +5회** (데이터가 데이터를 부르는 플라이휠).
  2. 오픈채팅 삼따봇과 연결: 봇 응답 말미에 "전체 답변은 삼따AI에서" 딥링크 → 채팅 트래픽을 AI로 회수.
  3. 답변 공유 카드(`/a/{id}`)를 카드뉴스형 이미지로 강화 → 단톡방 공유 최적화.
- **어떻게**: 게이트는 `ic_ai_usage`+`ai_bonus` 필드 재사용. 봇은 `_lib/chatbot.js` 응답 문자열에 링크 추가. 공유 OG는 기존 `/a/[id].js` 템플릿 개선.
- **측정**: AI 주 사용자 35→150, 사례 기여 월 20건+, 공유 경유 가입(ic_ref 쿠키) 월 30건.

### 2-3. 강의 마켓 오픈 (수익 레버 ①)

- **왜**: `ic_lectures` 테이블·API 있고 등록 0. 394명 커뮤니티 = 검증된 수강생 풀. 설계사 교육은 업계에서 돈이 도는 검증된 시장.
- **뭘**: 외부 강사(보상 실무·세무·리쿠르팅) 모집 → 플랫폼 수수료 15~20% 모델. 초기엔 삼백따리 네트워크 강사 3~5명으로 시작.
- **어떻게**: 신청·결제를 토스(1-1에서 가동)로 받고 `ic_purchases` 기록. 평점·후기는 Sprint 5 스펙 재사용. 다이제스트·홈 배너로 모객(자체 지면이라 광고비 0).
- **측정**: 분기 강의 6개, 강의당 수강 15명 × 5만원 × 수수료 20% = 강의당 15만 → 분기 90만.

### 2-4. 리쿠르팅 광고 정식 상품 (수익 레버 ②)

- **왜**: Phase 1에서 공고 20건+ 확보 시 지면 가치 발생. 리쿠르팅은 GA·조직장이 실제로 월 예산을 쓰는 영역(설계사 1명 위촉 가치가 크므로 광고 단가 수용도 높음).
- **뭘**: 공고 패키지 — 상단고정 7일 10만 / 메인배너 20만 / **월 구독 "채용 파트너" 30만**(공고 무제한+상단+다이제스트 노출+지원자 통계).
- **어떻게**: `postings/feature.js`+`feature-stats.js` 이미 존재. 구독은 토스 정기결제 또는 월 수동 갱신으로 시작. 지원자 통계는 기존 클릭 추적 확장.
- **측정**: 유료 공고 월 5건, 구독 파트너 3곳 → 월 100~150만.

### 2-5. 재방문 고착: 다이제스트 개인화 + 푸시 확대

- **왜**: 신규 308명/월이 들어와도 재방문 루프가 없으면 샌다. 푸시 9명은 방치 상태.
- **뭘**: 다이제스트에 "내가 즐겨찾은 보험사 새 소식"·"내 질문에 새 답변" 개인화 1블록. 푸시 옵트인 넛지를 가입 직후+AI 사용 직후 2지점에 배치.
- **어떻게**: 개인화는 즐겨찾기·질문 테이블 조인 1개씩. 푸시 넛지는 기존 `push/subscribe` 플로우에 트리거만 추가.
- **측정**: 주간 재방문율, 푸시 9→150, 다이제스트 클릭률 15%+.

## Phase 3 — 장기 플랫폼 확장 (2027~)

목표: **월 1,000만원+ · "설계사가 매일 여는 앱" → "설계사의 업무 OS".** 광고 의존에서 구독 매출로 축 이동.

### 3-1. 프리미엄 멤버십 (구독 매출의 시작)

- **뭘**: 월 9,900원 — 삼따AI 무제한 + 담보 비교 전체 + 사례 DB 검색 + 청구서류 일괄 다운로드 + 광고 제거.
- **왜 되나**: 사례 897건(계속 증가)+담보 366건은 대체재가 없다. 설계사에게 월 9,900원은 계약 1건의 극히 일부 — 가격 저항 낮음.
- **어떻게**: 토스 정기결제(빌링) + `role` 필드에 premium 등급(스키마 이미 지원). 무료 티어는 유지해 유입 깔때기 보존.
- **시뮬레이션**: MAU 800 중 전환 8% = 64명 × 9,900 = 월 63만 → MAU 2,000·전환 10%면 **월 198만**. 구독은 누적되는 매출이라는 점이 핵심.

### 3-2. CRM SaaS (ROADMAP Sprint 7 실행)

- **뭘**: 고객 DB·상담 단계·계약 진행·알림을 묶은 설계사용 경량 CRM. 개인 월 19,900원 / 조직장 대시보드(팀원 현황) 월 5~10만원.
- **왜 되나**: `ic_client_notes` 이미 존재 = 수요의 씨앗. 기존 업계 CRM은 무겁고 비싸다. "카톡으로 일하는 설계사"에 맞춘 경량형이 빈 자리. AFG 팀 조직을 베타 테스터로 쓸 수 있는 것이 남들에게 없는 이점.
- **어떻게**: Sprint 7 스펙 기반 단계 구축. 계약 만기·생일 알림은 카톡 메모 인프라 재사용. 조직장 대시보드가 실질 수익원(조직 단위 결제).
- **시뮬레이션**: 개인 50명 + 조직 10팀 = 월 150~200만.

### 3-3. B2B 광고·제휴 (업계의 미디어가 된다)

- **뭘**: 보험사·GA·인슈어테크 대상 월 단위 스폰서십(홈 배너+다이제스트+오픈채팅 공지 패키지, 월 100~300만원). 트래픽·전환 리포트 제공(admin 지표 인프라 재사용).
- **왜 되나**: "현직 설계사 수천 명이 매일 보는 지면"은 보험업계 B2B 마케터가 정확히 원하는 것. MAU 2,000+ 도달 시 영업 가능.
- **조건**: MAU 2,000 · 다이제스트 도달 1,000 이상 확보 후 개시. 그 전에 팔면 단가만 망가진다.

### 3-4. 커뮤니티 사업 확장 (삼백따리 × 인슈어커넥트)

- **뭘**: ① 지역 정기모임 정례화 → 모임 광고·스폰서 매출과 직결 ② 연 1회 오프라인 컨퍼런스(부스+티켓) ③ 파트너 네트워크(손사·세무사·변호사) 유료 등록 디렉토리.
- **왜 되나**: 커뮤니티는 분위기가 자산 — 온라인 지표가 아니라 "저기 가면 사람이 있다"는 인식이 B2B 단가를 만든다. 모임 광고가 첫 주문 5건을 만든 것이 증거.
- **어떻게**: meetings 인프라+ad_orders 재사용. 컨퍼런스는 Phase 3 하반기, MAU 2,000 이후.

## 4. 실행 관리

### 우선순위 원칙 (막히면 이 순서로 판단)

1. **결제 마찰 제거 > 신규 상품** — 팔리는 걸 못 받는 게 최악.
2. **검증된 수요(모임 광고·네이버 전산 유입)에 먼저 붙인다.**
3. **카톡 254명 > 푸시 9명** — 도달 큰 채널부터.
4. 지면(공고·강의)이 비었으면 **공급 확보가 유료화보다 먼저.**
5. B2B 단가는 MAU가 만든다 — 조급하게 팔지 않는다.
